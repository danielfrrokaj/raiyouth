# Concept: Money Island Onboarding

Instead of making onboarding feel like form-filling, the user feels like they are **building their own Money Island**.

Each onboarding section unlocks a new part of the island. The app still collects the same required data, but the experience feels like progress, not paperwork.

The user journey becomes:

> “I’m building my first banking world.”

Not:

> “I’m completing KYC forms.”

---

# Main experience

The onboarding has a small island preview at the top of each screen.

At the beginning, the island is hidden in clouds. As the user completes steps, the island grows:

1. Island appears
2. Guide character appears
3. Card forge is built
4. Signal tower lights up
5. Profile house appears
6. Trust gate opens
7. Home base is marked
8. Vault is locked
9. Island is complete

This gives the user a visible sense of progress.

---

# Recommended flow

## 1. Choose Your Quest

**Data collected:**

* Intent
* Persona

**User-facing idea:**

The user chooses why they are joining.

Examples:

* Save for something cool
* Spend smarter
* Learn money skills
* Get my first card

Based on the choice, the app assigns a persona:

* Goal Getter
* Smart Spender
* Explorer
* Card Starter

**Game moment:**

The island appears from the clouds.

**Reward message:**

> Island Unlocked
> Your money journey has started.

---

## 2. Meet Your Money Guide

**Data used:**

* Persona

**User-facing idea:**

The user gets a guide based on their money style.

Example:

> You’re a Goal Getter.
> You like setting targets and working toward them.

The guide is not just decorative. It can appear throughout onboarding with small encouraging messages.

**Game moment:**

A small guide character appears on the island.

**Reward message:**

> Guide Unlocked
> Your money guide is ready.

---

## 3. Card Forge

**Data collected:**

* Card Color
* Alias Name

**User-facing idea:**

Instead of “customize card,” the user is “forging” their first money card.

They choose:

* Card color
* Alias or display name

**Game moment:**

A small card workshop appears on the island. The card is revealed with a glow or sparkle.

**Reward message:**

> Card Forge Unlocked
> Your first card style is ready.

This should be one of the most visual and fun steps.

---

## 4. Signal Tower

**Data collected:**

* Phone Number
* SMS OTP
* Email Address

**User-facing idea:**

The user builds a Signal Tower to protect and recover their account.

Phone number builds the tower.
OTP activates it.
Email adds a backup beacon.

**Game moment:**

The tower lights up after OTP verification.

**Reward messages:**

After phone:

> Signal Tower Built
> Your phone has been added.

After OTP:

> Signal Tower Activated
> Your phone is verified.

After email:

> Backup Beacon Added
> Your recovery contact is ready.

This makes security feel useful rather than annoying.

---

## 5. Profile House

**Data collected:**

* First Name
* Last Name
* Alias Name
* Date of Birth

**User-facing idea:**

The user creates their profile house on the island.

The house represents their identity inside the app. The alias can appear on a small sign outside the house.

**Game moment:**

A profile house appears with the user’s alias.

**Reward message:**

> Profile House Built
> Your profile is ready.

Date of birth should remain clear and serious:

> We need your birth date to set up the right account for you.

---

## 6. Trust Gate

**Data collected:**

* Country of Residence
* Citizenship Status
* Verification Method
* Selected Document Type
* ID Number
* Biometric Selfie

**User-facing idea:**

This is the identity verification section. It should feel secure, premium, and calm.

The user is opening the Trust Gate by confirming who they are.

Sub-steps:

* Confirm country
* Confirm citizenship
* Choose verification method
* Choose document
* Scan document
* Complete face match

**Game moment:**

A locked gate gradually opens as the user completes verification.

**Reward message:**

> Trust Gate Opened
> Your identity checkpoint is complete.

Important: this part should not feel silly or too childish. It should feel like a secure checkpoint.

---

## 7. Home Base

**Data collected:**

* Street Name
* Floor & Unit
* Zip Code

**User-facing idea:**

The user sets their real-world Home Base.

This makes address collection feel softer while still being transparent.

**Game moment:**

A location pin appears next to the island, connecting the island to the real world.

**Reward message:**

> Home Base Set
> Your residential address has been added.

---

## 8. Vault Lock

**Data collected:**

* Passcode

**User-facing idea:**

The user creates the secret key to lock their app.

This is a strong final security moment.

**Game moment:**

A vault appears on the island and locks.

**Reward message:**

> Vault Locked
> Your app is protected.

---

## 9. Island Complete

**Data used:**

* All onboarding data

**User-facing idea:**

The final screen shows the completed island with all unlocked pieces.

Show a summary like:

> You’re ready, Alex.
> Your Money Island is complete.

Then list:

* Quest selected
* Card created
* Signal Tower active
* Profile ready
* Trust Gate open
* Home Base set
* Vault locked

CTA:

> Enter My App

# Visual direction

The style should feel:

> Premium, playful, clean, and safe.

Good references:

* Duolingo progress energy
* Apple Wallet polish
* Monument Valley-style floating world
* Soft 3D icons
* Gentle gradients
* Friendly but not childish

Avoid:

* Babyish cartoons
* Too many coins
* Loud confetti on every step
* Making ID verification feel like a joke
* Overcomplicated game mechanics

---

# Why this works

The Money Island concept works because every data category has a natural game metaphor:

| App data            | Game metaphor   |
| ------------------- | --------------- |
| Intent / Persona    | Quest and guide |
| Card color / Alias  | Card forge      |
| Phone / OTP / Email | Signal tower    |
| Name / DOB          | Profile house   |
| KYC / ID / Selfie   | Trust gate      |
| Address             | Home base       |
| Passcode            | Vault lock      |

The user always understands why they are entering information because each step has a purpose inside the story.

---

# Core product message

The app should not say:

> Complete onboarding.

It should say:

> Build your Money Island.

The user is not just signing up. They are creating their first safe money space.
